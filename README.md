# DSPy Story Pipeline

A DSPy pipeline that turns a one-line idea into a complete, review-ready video
**story package** — idea, theme, story structure, raised stakes, scripts in four
formats, plus a title/thumbnail concept and a YouTube description. Every stage is
generated by one model and graded by an **independent** model on a different
provider, so the judge can never inflate its own work.

## How it works

The engine is generic over a **stage**. Each stage runs the same loop
([pipeline/orchestrator.py](pipeline/orchestrator.py)):

1. **Gate** — generate a candidate, grade it; reject and regenerate up to
   `MAX_GEN_ATTEMPTS` (5) until one passes. On pass it gets a story id `ST-####`.
2. **Score + iterate** — improve the passing artifact up to `MAX_ITER` (3) times,
   toward `TARGET_SCORE` (95).
3. **Finalize** — reached target → `ready_for_review`; otherwise **parked**
   (still saved, just below the bar). Nothing generated is ever deleted.

**The #1 rule:** the evaluator runs on a *different provider/model* than the
generator (enforced in `assert_separation`). Today:

- **Generator / iterator** → Z.ai GLM (`ZAI_API_KEY`)
- **Evaluator / reevaluator** → OpenRouter (`OPENROUTER_API_KEY`)

Change either in [config.py](config.py).

## The pipeline

```
idea → theme → story → stakebake → script ───────────┐
                                    script_long ──────┤
                                    script_screenplay ┤   (all four formats)
                                    script_podcast ───┘
                                          │ (promote the long-form script)
                                          ├─▶ packaging  (title + thumbnail)
                                          └─▶ description (YouTube description)
```

The spine (idea→stakebake) auto-promotes each stage on target and stops at the
first weak link. All four script formats are produced off the accepted stakebake;
**packaging and description are built from the finished long-form script** (it
holds the whole story). Every artifact of one video shares a single `ST-####`,
while the formats stay separable by stage.

## Setup

Requires Python 3.11.

```bash
python -m venv .venv
.venv/Scripts/python -m pip install -r requirements.txt   # Windows
# cp .env.example .env  -> fill in ZAI_API_KEY and OPENROUTER_API_KEY
```

## Usage

```bash
python run.py new-channel demo-sports                       # scaffold channels/demo-sports/
# add ideas (one plain-text idea per line) to channels/demo-sports/seeds/briefs.jsonl
python run.py chain --channel demo-sports                   # run the whole pipeline over the seed queue
python run.py chain --channel demo-sports --brief "..."     # run one brief
python run.py chain --channel demo-sports --brief "..." --dry-run   # offline plumbing test, no keys
python run.py generate --channel demo-sports --stage idea --brief "..."   # one stage, live
python run.py manual   --channel demo-sports --stage idea --file manual_input.json  # hand-authored, no keys
python run.py route    --channel demo-sports               # apply review/human_review.csv decisions
python run.py learn    --channel demo-sports --stage idea  # retrain that stage's generator from accepted examples
python run.py package  --channel demo-sports               # assemble a video's deliverables into one doc
```

**Modes:** *live* (real API calls), *dry-run* (offline stub LMs, no keys —
exercises the wiring), and *manual* (your own content + scores through the real
engine).

## Where things land

Per channel, under `channels/<name>/`:

- `outputs/{candidates,accepted,rejected,parked,archived}/` — versioned `.json` + readable `.txt`
- `logs/` — one narrative log per run · `metrics/scores.jsonl` — one row per scored version
- `memory/` — `trainset.jsonl` (accepted examples), `compiled/<stage>_generator.json` (learned), `run_index.jsonl`
- `review/human_review.csv` — your accept/reject/revise decisions for `route`
